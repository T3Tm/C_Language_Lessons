함수 , 전역변수(scope)

1 - 함수

함수란: 입력을 받아서 특정한 작업을 수행해서 결과를 반환하는 것이다!

함수가 필요한 이유:동일한 작업을 사전에 방지하기 위해서이다.

어찌보면 반복문과 비슷하다고 생각할 수 있다.

반복이 되는 것을 방지하는 것은 같지만 반복문은 여러 번 사용할 수는 없다.

하지만 함수는 코드들은 하나로 묶어서 여러 번 재사용할 수 있다.

그리고 개발할 때 과정이 쉬워지고 체계적이게 되며 유지보수도 쉬워진다.

함수에는 2가지의 종류가 있다.
1. 사용자 정의 함수
2. 라이브러리 함수(stdio.h)에 들어있는 것들을 라이브러리 함수라고 한다!


그럼 함수 정의를 보자

함수는 다음과 같이 기술한다.
반환 자료형 함수이름(매개 변수..){
    문장;
    return 반환에 맞는 자료형;
}

함수는 자신이 원하는 형태로 만들어서 필요할 때마다 갖다 쓰면 된다.

예를 들어) 방금 실습했던 별을 찍는 코드를 함수에 넣고
함수를 호출하여 화면에 두 번 별을 찍어보자.

함수 원형(function prototyping)

컴파일러에게 함수에 대하여 미리 알리는 것

나중에 정의해도 에러가 안난다!
#include stdlib.h 각종 다 있는 라이브러리 , 대체적으로 랜덤을 주로 사용할 때 부름
time.h < 시간 관련 라이브러리
rand() 의 범위는  0 ~ 32767 이다.

변수는 범위(scope), 생존 시간(lifetime),연결(linkage)
범위 : 변수가 사용가능한 범위
생존 시간: 메모리에 존재하는 시간
연결 : 다른 영역에 있는 변수와의 연결 상태


변수-지역 변수, 전역 변수
함수 내부에서 정의하면 지역변수, 외부에서 정의하면 전역변수


지역 변수는 선언된 블록을 떠나면 사용이 불가능해지고 메모리에서 해제된다.

지역이 다를 때에는 변수 이름을 같게 설정 가능하다.

매개 변수도 지역변수이다.



전역 변수의 초기값은 0


생존 기간:
-정적 할당:
    프로그램 실행 기간 동안 계속 유지
-동적 할당:
    블록에 들어갈 때 생성
    블록에서 나올 떄 소멸

생존 기간을 결정하는 요인:
    변수가 선언된 위치
    저장 유형 지정자

저장 유형 지정자:
    1. auto
    2. register
    3. static
    4. extern

auto: 블록이 선언한 위치에서 자동으로 만들어지고 블록을 벗어나게 되며 
자동으로 소멸된다.

지역변수는 auto가 생략된 것이다.

static은 정적 변수로 프로그램 실행 기간 동안 계속 유지 되기에 다음 예시에서 결과가 
다르게 나온다. 
void sub(){
    static int scount = 0;
    int account = 0;

}

register: cput안에 있는 register에 변수가 저장된다.

volatile 지정자는 하드웨어가 수시로 변수의 값을 변경하는 경우 사용

연결: 다른 범위에 속하는 변수들을 서로 연결하는 것
    1. 외부연결 
    2. 내부 연결
    3. 무연결

전역 변수만이 연결을 가질 수 있다.

외부 연결:
전역 변수에 extern을 붙여서 서로 연결하는 것이다.
